# EmployeeDirectoryManagerApplication
Reflection: “What did you learn about data persistence and how it affects the reliability of real-world applications?”
Data persistence is extremely important in real-life situations. Without it, digital apps and means to record information
on computers would be obsolete and useless, as it would not pay to have to write in all of the data every time you
loaded the app or computer. With the ability to save and load from saved files, users can save their data and reload it when
starting the app from fresh. Real-world applications are incredibly more reliable when a proper and robust saving system
is implemented and the user can rely on that the program is saving their data in a secure and easy-to-find location.
This is the reason why I always save my test files to the Desktop, as it gives me an easy way to check if the file saved
correctly during debugging.
